Acumatica Customization 101

John Kihiu

Software Developer

|
January 15, 2023
|
5 min read

In this comprehensive guide, we'll explore the fundamentals of customizing Acumatica ERP using C# and .NET. Whether you're a developer new to Acumatica or looking to expand your skills, this tutorial will provide you with the foundational knowledge needed to build robust customizations.

Getting Started with Acumatica Customization

Before diving into customization, ensure you have the following prerequisites in place:

  • Acumatica ERP installed and configured
  • Visual Studio 2019 or later
  • Acumatica Framework SDK
  • Basic understanding of C# and .NET

Creating Your First Customization Project

Let's create a simple custom project that demonstrates the core concepts of Acumatica customization:

using PX.Data;
using PX.Objects.AR;

namespace Acumatica.Customization
{
    public class ARInvoiceEntryExt : PXGraphExtension<ARInvoiceEntry>
    {
        public PXAction<ARInvoice> MyCustomAction;
        
        [PXButton]
        [PXUIField(DisplayName = "Custom Action")]
        protected void myCustomAction()
        {
            var invoice = Base.Document.Current;
            // Your custom logic here
            Base.Actions.PressSave();
        }
    }
}

Understanding the Code Structure

Let's break down each component of the customization:

  1. PXGraphExtension - Extends an existing Acumatica graph
  2. PXAction - Creates a custom button/action
  3. PXUIField - Configures the UI display properties

Deploying Your Customization

After creating your customizations, follow these steps to deploy them:

  1. Build your solution in Release mode
  2. Copy the DLL to the Acumatica bin folder
  3. Restart the Acumatica website
  4. Enable the customization in the Customization Project Editor
"Acumatica's extensibility framework allows developers to add powerful functionality without modifying the core system, ensuring easy upgrades and maintainability."

Best Practices

  • Always usePXCacheExtension for adding fields to existing DACs
  • Implement proper error handling and logging
  • Write unit tests for your custom logic
  • Document your customizations thoroughly
