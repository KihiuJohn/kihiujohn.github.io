UDFs vs Custom DACs in Acumatica — When to Use Which

When to use Acumatica User-Defined Fields, when to declare a custom DAC, and the tradeoffs that determine which path you should take on a real project.

John KihiuMay 24, 20266 min read

"Should this be a UDF or its own DAC?" is a question that comes up on every Acumatica project. Here is the decision matrix I walk clients through.

Use a UDF when

  • You are adding one or two fields to an existing screen.
  • No reporting, automation, or workflow is needed for the new field.
  • End users should be able to add the field without developer involvement.

Use a custom DAC when

  • The new entity has its own workflow, validation, or business rules.
  • You need a new screen or a new tab on an existing screen.
  • You need to report on the data in complex ways.
  • You need to expose the data via the API.
UDFs are first-class in Modern UI Since 2025 R2, UDFs no longer live in a separate "User-Defined Fields" tab. They can be added to any section via Screen Configuration. They are a real option, not a workaround.
